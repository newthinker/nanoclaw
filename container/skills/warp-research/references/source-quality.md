# 来源质量与交叉验证

## 可信度 5 级(高 → 低)

1. 官方文档 / 一手数据(发布方自己的文档、原始数据集)
2. 作者原文 / 原始论文(一手作者、同行评审)
3. 知名分析机构(有信誉的研究/媒体)
4. 社区讨论(HN / Reddit / 论坛 / 博客)
5. 匿名 / SEO 水文 / 营销软文 —— **不计作独立源**

## 交叉验证:2+ 独立源规则

- "**独立**" = 不同机构/作者/平台,且**非互相转载同一原始来源**。要溯源:转载不算第二源。
- 关键结论(影响最终建议/立场的)必须 2+ 独立源支撑。

## 置信度标记(报告内统一用)

| 标记 | 含义 |
|------|------|
| ✅ | 2+ 独立一手源一致 |
| ℹ️ | 1 一手 + 1 二手,或 2 二手一致 |
| ⚠️ | 单源,或源间矛盾 |
| ❓ | 仅推测/无来源 |

## 时效衰减(过期要标"截至 X 时点")

- 技术:6 个月 · 市场:3 个月 · 基本面/财务:1 个月。

## 矛盾标注格式

```markdown
关于 <议题> 存在分歧:
- 来源A(<级别>, <日期>):<说法>
- 来源B(<级别>, <日期>):<说法>
判断:<采信哪方及理由 / 或并陈> ⚠️
```

## 低质量识别(出现即不作独立源计数)

无作者无日期、洗稿/转载、过度 SEO 关键词堆砌、纯营销话术。
