# 停止判据 — 何时调研够了

调研最大的失败模式是**无限搜索**。本文件给出"已足够"的判据和**硬上限**。

## "够判断就停"的信号

- 再搜只是重复已知信息(边际收益趋零)。
- 已能给出**带明确立场**的 TL;DR。
- 关键结论已满足下表对应深度档的交叉验证要求。

## 三档深度的"已足够"判据

| 深度 | 已足够 | 交叉验证要求 |
|------|--------|--------------|
| **overview** | 每个核心子问题各有 1 个可信来源已答;3–4 题覆盖 | 不强制 |
| **standard**(默认) | 每个 priority≥中 的子问题有答案;5–7 题覆盖;矛盾已标注 | 关键结论 2+ 独立源 |
| **deep** | 全部子问题已答;正反观点均覆盖;局限性可明确写出 | 关键结论一手来源交叉验证 |

## 防无限调研硬上限(写死,务必遵守)

- 单子问题搜索轮次 **≤ 3**;单题 web 调用 **≤ 6**。
- 整体 web 调用总数:overview **≤ 15** / standard **≤ 30** / deep **≤ 50**。
- 子问题树总数 **≤ 10**(动态新增不得突破)。
- 触发范围蔓延(题数 > 10 或调用超预算 2×)→ 回到 `motivation` 校准,砍掉无关分支;
  **宁可对缺口标 `info-gap` 也不无限搜**。

## 降级出口

任一硬上限触顶 → 停止采集,带着已有信息进入分析/成稿,在报告"局限性"里写明哪些子问题是 `info-gap`。
